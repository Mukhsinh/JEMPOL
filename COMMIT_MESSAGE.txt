fix: Perbaikan form submission QR code - validasi dan error handling

Mengatasi error 500 dan 404 pada form submission dari QR code untuk:
- Tiket Internal
- Tiket Eksternal
- Survey Kepuasan

## Root Cause
- Frontend tidak mengirim semua field yang diperlukan (reporter_department, reporter_position)
- Tidak ada validasi unit_id sebelum submit
- Tidak ada handling untuk non-JSON response atau error 404

## Changes

### Frontend Fixes
- PublicInternalTicket.tsx:
  * Tambah validasi unit_id (format UUID) sebelum submit
  * Tambah pengiriman reporter_department & reporter_position
  * Tambah validasi response content-type
  * Tambah error handling untuk 404 dan non-JSON response
  * Tambah logging lengkap untuk debugging

### Testing Infrastructure
- test-qr-forms-integration.js:
  * Script test otomatis untuk semua 3 endpoint
  * Auto-fetch valid unit_id dari database
  * Comprehensive error reporting
