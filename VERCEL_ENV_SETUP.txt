PANDUAN SETUP ENVIRONMENT VARIABLES DI VERCEL
==============================================

Untuk menjalankan aplikasi di Vercel dengan benar, Anda perlu mengatur environment variables berikut di dashboard Vercel:

1. Login ke Vercel Dashboard (https://vercel.com)
2. Pilih project Anda
3. Pergi ke Settings > Environment Variables
4. Tambahkan variable berikut:

WAJIB:
------
VITE_SUPABASE_URL=https://jxxzbdivafzzwqhagwrf.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6Imp4eHpiZGl2YWZ6endxaGFnd3JmIiwicm9sZSI6ImFub24iLCJpYXQiOjE3NjQ5MTkwNTEsImV4cCI6MjA4MDQ5NTA1MX0.ICOtGuxrD19GtawdR9JAsnFn9XsHxWkr1aHCEkgHqXg

OPSIONAL (jika ingin menggunakan backend terpisah):
---------------------------------------------------
VITE_API_URL=https://your-backend-url.com/api
VITE_FRONTEND_URL=https://your-vercel-app.vercel.app

CATATAN PENTING:
----------------
1. Aplikasi sekarang sudah menggunakan fallback ke Supabase langsung jika backend tidak tersedia
2. Untuk production, disarankan deploy backend ke platform terpisah seperti:
   - Railway (https://railway.app)
   - Render (https://render.com)
   - Fly.io (https://fly.io)
   - Heroku (https://heroku.com)

3. Setelah deploy backend, update VITE_API_URL di Vercel environment variables

CARA DEPLOY BACKEND (OPSIONAL):
--------------------------------
1. Pilih platform hosting (contoh: Railway)
2. Connect repository GitHub Anda
3. Set root directory ke "backend"
4. Set build command: npm install
5. Set start command: npm start
6. Tambahkan environment variables backend:
   - PORT=3004
   - SUPABASE_URL=https://jxxzbdivafzzwqhagwrf.supabase.co
   - SUPABASE_SERVICE_ROLE_KEY=[your-service-role-key]
   - FRONTEND_URL=https://your-vercel-app.vercel.app
   - NODE_ENV=production

7. Deploy dan copy URL backend
8. Update VITE_API_URL di Vercel dengan URL backend tersebut
9. Redeploy frontend di Vercel

TESTING:
--------
Setelah setup, test aplikasi dengan:
1. Buka form tiket eksternal
2. Submit tiket
3. Cek console browser untuk melihat apakah menggunakan backend atau fallback ke Supabase
4. Verifikasi tiket masuk ke database Supabase

Jika ada error, cek:
- Browser console untuk error detail
- Vercel deployment logs
- Supabase logs (jika menggunakan fallback)
